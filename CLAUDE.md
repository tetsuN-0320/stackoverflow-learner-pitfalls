## コーディング上の方針

### 言語・スタイル
- **Python バージョン**: 3.11 以上
- **コードフォーマッタ**: black
- **リンター**: ruff
- **型ヒント**: 関数の引数・戻り値には可能な限り付与（mypy対応）
- **docstring**: Google スタイル（日本語可）
- **コメント**: 日本語OK。なぜそうするかの「why」を残す
- **変数名・関数名**: 英語スネークケース（`fetch_stackex_questions` 等）
- **クラス名**: 英語パスカルケース（`StackExClient` 等）

### 設計原則
- `src/` と `notebooks/` は分離する。再利用ロジックは必ず `src/` に置き、ノートブックは `from src.xxx import yyy` で import
- 3層分析（マクロ・メゾ・ミクロ）を `src/analysis/` 配下のモジュールで責務分離
- Stack Exchange API のレート制限（バックオフ含む）を厳守。tenacity で自動制御
- データ取得は SQLite キャッシュで再実行コストを抑える
- 対象言語は `config/target_languages.yml` で外出し、後から追加可能に
- 環境変数（API キー）は `.env` で管理し、絶対にコミットしない
- フロント用JSONはネットワーク図を言語別ファイルに分割し、動的ロード

### テスト
- 主要な関数には `pytest` でユニットテストを追加
- API クライアントはモックでテスト、実APIアクセスは `pytest -m integration` で分離

### Stack Exchange API利用上の必須事項
- レート制限: 認証あり10,000req/日を厳守
- API側から `backoff` パラメータが返る場合、その秒数だけ待機する
- データは [Stack Exchange Network のCC-BY-SA](https://stackoverflow.com/legal/terms-of-service/public) に基づき利用。出典明記必須
- About ページで「Stack Overflow / Stack Exchange の規約に基づきデータを利用」を明記

### D3.js 実装方針（重要）
- **ゼロから自作しない**。既存サンプル（[Observable](https://observablehq.com/) 等）から派生する形で実装
- ネットワーク図は force-directed graph（典型的な「ノードとエッジ」のグラフ）
- ノード数は上位50〜100個に絞り、視認性を確保
- ホバーで詳細表示、クリックで関連質問例を表示

## 3層分析の構造（実装計画書より抜粋）

第3作の分析は以下の3層で構成する。Claude Code は実装中、この階層構造を意識して進める。

### マクロ層（言語の盛衰）
- 言語別タグの質問数推移（過去5〜10年）
- Python / JavaScript / Java / Go の4言語を中心に
- Developer Survey の「使用言語」と突合

### メゾ層（タグの組み合わせ）
- タグ共起ネットワーク（`networkx` で構築）
- Louvain法によるコミュニティ検出
- 中心性指標で「学習領域の中核タグ」を特定

### ミクロ層（質問本文）
- TF-IDF + HDBSCAN による質問のトピッククラスタリング
- 「Pythonでpandasを学ぶ人がつまづく上位5パターン」のような具体的発見
- エラーメッセージの頻出語抽出

最終的に「マクロでの言語選択 → メゾでの学習領域 → ミクロでの具体的つまづき」という **学習ロードマップへの示唆** に繋げる。
